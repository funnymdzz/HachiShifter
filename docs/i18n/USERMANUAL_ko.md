# HiFiShifter 사용자 매뉴얼

[简体中文](USERMANUAL.md) | [繁體中文](USERMANUAL_zh-TW.md) | [English](USERMANUAL_en.md) | [日本語](USERMANUAL_ja.md) | [한국어](USERMANUAL_ko.md)

HiFiShifter는 그래픽 보컬 편집 및 합성 도구입니다. 멀티트랙 오디오 클립 처리를 지원하며, 트랙 그룹 단위로 여러 보코더를 사용하여 보컬의 피치 보정 및 파라미터 조정을 수행하여 인간 VOCALOID 제작의 편집과 튜닝을 통합합니다.

## 1. 설치

사용자의 운영 체제 및 아키텍처에 맞는 HiFiShifter 설치 패키지를 다운로드하세요. OS별로 `Windows`, `macOS`, `Linux` 세 가지, 아키텍처별로 `x86_64`와 `arm64` 두 가지가 있습니다.

- Windows 버전은 NSIS 설치 프로그램(`installer`)과 포터블 압축 파일(`portable`) 두 가지를 제공합니다. 일반 사용자는 설치 프로그램 버전을 선택하여 설치하면 됩니다.  
  Windows 사용자로서 `x86_64`와 `arm64`의 차이를 모르신다면 `x86_64` 버전을 선택하세요. `arm64`가 무엇인지 잘 알고 있고 실제로 Windows ARM 장치를 보유하고 있는 경우에만 `arm64` 버전을 다운로드하여 사용하세요.

- macOS 버전은 서명되지 않은 dmg 설치 패키지를 제공합니다. 아직 서명되지 않았기 때문에 설치 시 약간 복잡한 절차를 거쳐 설치를 허용해야 합니다.  
  M 칩 macOS 사용자는 `arm64` 버전을 설치하세요. 오래된 Intel 사용자만 `x86_64` 버전이 필요합니다.  
  dmg를 더블 클릭한 후 "파일이 손상되었습니다"라는 메시지가 표시되면 다음 단계를 따르세요:
    1. 터미널에서 `xattr -cr /Applications/HiFiShifter.app` 입력；
    2. `시스템 설정` -> `개인정보 보호 및 보안` -> `그래도 열기`를 클릭하여 앱 실행을 허용합니다.

- Linux 버전은 AppImage 패키지를 제공합니다. AppImage 패키지는 파일의 `속성 -> 권한`에서 `파일을 프로그램으로 실행 허용`을 체크한 후 직접 실행할 수 있습니다.

**WebView 정보**: HiFiShifter 는 Rust + Tauri 프레임워크로 구축되었으며, 인터페이스를 표시하려면 WebView 구성 요소가 필요합니다.

- **Windows**: Edge WebView2 가 필요합니다. Windows 10 (버전 1803 이상) 및 Windows 11 에는 사전 설치되어 있으므로 추가 작업이 필요하지 않습니다. 이전 버전의 Windows를 사용하거나 구성 요소가 없는 경우 설치 프로그램이 자동으로 다운로드하라는 메시지를 표시합니다. 자세한 내용은 [Tauri 공식 문서](https://tauri.app/ko/start/prerequisites/#webview2)를 참조하세요. 일반 사용자는 설치 프로그램을 실행하기만 하면 됩니다.
- **macOS**: 시스템에 WebKit이 포함되어 있으므로 추가 설치가 필요하지 않습니다.
- **Linux**: WebKitGTK 가 필요합니다. 대부분의 주요 배포판(Ubuntu, Fedora, Arch Linux 등)에는 기본적으로 포함되어 있습니다. 누락된 경우 패키지 관리자를 사용하여 `webkit2gtk` 를 설치하세요(예: `sudo apt install webkit2gtk`). 자세한 내용은 해당 배포판의 문서를 참조하세요.

## 2. 메뉴

`파일` 메뉴에서는 HiFiShifter 프로젝트 파일을 열고 저장할 수 있으며, 오디오 파일 가져오기, Reaper 프로젝트(`.rpp`) 가져오기, VocalShifter 프로젝트(`.vshp` 또는 `.vsp`) 가져오기, MIDI 파일 가져오기, 오디오 내보내기 등의 작업을 할 수 있습니다.

HiFiShifter 프로젝트 파일의 확장자는 `.hshp` 또는 `.hsp`입니다. 그 외에도 `다른 이름으로 저장` 기능은 프로젝트 파일을 일반 텍스트 `json` 파일로 저장하거나, 현재 프로젝트와 사용된 모든 미디어 파일을 함께 아카이브 압축 파일 `.zip`으로 패키징할 수 있습니다. 현재 HiFiShifter는 일반 오디오 파일만 가져올 수 있으며 비디오 파일은 지원하지 않습니다.

자동 백업 기능을 통해 프로젝트 파일의 자동 백업을 설정할 수 있습니다. `저장 시 백업`과 `정기 백업`의 두 가지 모드가 있습니다.

- `저장 시 백업`: 프로젝트 파일을 덮어쓰기 저장할 때마다 이전 프로젝트 파일이 자동으로 원래 파일명에 `-bak`가 추가된 백업 파일(예: `.hshp-bak`, `.hsp-bak`)로 이름이 변경됩니다. 기본값으로 활성화되어 있습니다.
- `정기 백업`: 편집 중에 설정한 간격과 경로에 따라 자동으로 백업 프로젝트 파일을 생성합니다. 기본값으로 비활성화되어 있습니다.

`편집` 메뉴에서는 다양한 편집 작업을 할 수 있습니다. 일반적인 트랙 편집 및 파라미터 편집 외에도 `Reaper 클립보드 데이터 붙여넣기`와 `VocalShifter 클립보드 데이터 붙여넣기` 두 항목이 있습니다.

- **Reaper 클립보드 데이터 붙여넣기**: Reaper에서 아이템(Item) 복사, 트랙 복사, MIDI 노트 복사 작업을 수행한 후, 이 기능을 통해 Reaper 클립보드 데이터를 빠르게 HiFiShifter로 가져올 수 있습니다.
    - 아이템 데이터: HiFiShifter의 노트 클립으로 가져오며, Reaper의 튜닝 데이터(전체 튜닝 및 피치 엔벨로프 모두)를 유지합니다.
    - 트랙 데이터: 트랙과 그 안의 아이템을 HiFiShifter의 트랙 및 오디오 클립으로 가져옵니다. 트랙 그룹을 유지할 수 있습니다.
    - MIDI 노트 데이터: 다른 DAW(Reaper, FL Studio 등)에서 노트 데이터를 클립보드 MIDI 노트 데이터로 내보낸 뒤, HiFiShifter의 파라미터 에디터에서 `선택` 도구로 피치 곡선 구간을 선택하면 해당 구간에 클립보드 MIDI 노트 데이터를 가져올 수 있습니다. MIDI 가져오기 기능에 대한 자세한 설명은 [음높이 참조 클립](#음높이-참조-클립) 섹션을 참고하세요.

- **VocalShifter 클립보드 데이터 붙여넣기**: VocalShifter 또는 VocalShifter LE에서 파라미터 라인 복사, 오디오 클립 복사, 트랙 복사 작업을 수행한 후, 이 기능을 통해 VocalShifter 클립보드 데이터를 빠르게 HiFiShifter로 가져올 수 있습니다.
    - 파라미터 라인 데이터: HiFiShifter의 파라미터 에디터에서 `선택` 도구로 파라미터 라인 구간을 선택한 후, VocalShifter 클립보드의 파라미터 라인 데이터를 해당 구간으로 가져올 수 있습니다.
    - 오디오 클립 데이터: HiFiShifter의 노트 클립으로 가져오며, 다양한 파라미터 라인 데이터를 유지합니다.
    - 트랙 데이터: 트랙과 그 안의 오디오 클립을 HiFiShifter의 트랙으로 가져옵니다. HiFiShifter는 현재 마지막으로 복사한 내용이 오디오 클립인지 트랙인지 구분할 수 없습니다. 트랙을 가져오려는 경우, VocalShifter에서 트랙 복사 작업을 수행하기 전에 VocalShifter 프로젝트에서 선택된 오디오 클립이 없는지 확인하세요. 그렇지 않으면 선택된 오디오 클립만 가져와집니다.

`스트레치` 메뉴에서는 현재 프로젝트와 전역 스트레치 알고리즘을 수정할 수 있습니다.

## 3. 트랙 인터페이스

소프트웨어의 대략적인 작동 로직과 단축키는 Reaper, VocalShifter, VEGAS Pro 등의 DAW 소프트웨어를 참고할 수 있습니다. `보기 -> 키보드 단축키`에서 자신의 단축키 선호도를 조정할 수 있습니다. 아래 설명은 모두 기본 단축키를 기준으로 합니다.

트랙 인터페이스는 HiFiShifter의 핵심 기능 중 하나로, 오디오 클립을 자르고 붙이는 등의 작업을 할 수 있습니다. 작동 로직은 기본적으로 Reaper를 참고할 수 있습니다.

인터페이스 조작은 마우스 가운데 버튼(마우스 휠 누름)을 드래그하여 화면을 이동할 수 있습니다. 수평/수직 확대/축소 또는 스크롤은 `Ctrl`, `Alt`, `Shift` 등의 수정 키를 누른 상태에서 마우스 휠을 굴려 조정할 수 있습니다. 이러한 수정 키는 단축키 설정에서 조정할 수 있습니다.

자주 사용하는 단축키:

- `스페이스바`: 재생 / 일시 정지 (재생 시작점으로 돌아가지 않음)
- `Enter`: 재생 / 정지 (재생 시작점으로 돌아감)
- `S`: 분할
- `G`: 그룹
- `U`: 그룹 해제
- `Ctrl + C`: 복사
- `Ctrl + V`: 붙여넣기
- `Ctrl + Z`: 실행 취소
- `Ctrl + Y`: 다시 실행
- `Ctrl + A`: 모두 선택
- `Ctrl + R`: 선택 해제
- `Delete`: 오디오 클립 삭제
- `-` / `=`: 선택한 오디오 클립의 파라미터 라인을 아래로/위로 이동
- 수정 키 `Alt`: 누른 상태에서 클립 시작/끝을 드래그하면 클립을 스트레치하고, 클립 중간을 드래그하면 내부 오프셋(슬립 편집)을 조정합니다.
- 수정 키 `Shift`: 누른 상태에서 그리드 스냅을 일시적으로 전환합니다.
- 수정 키 `Ctrl`: 누른 상태에서 클립을 드래그하면 복사합니다.

클립 왼쪽 상단의 작은 원은 볼륨 조절 노브이고, `M` 버튼은 해당 클립을 개별적으로 음소거하며, `F` 버튼은 해당 클립의 포먼트 편집 메뉴를 엽니다. 클립의 왼쪽/오른쪽 끝은 페이드 인/페이드 아웃 길이를 조정할 수 있습니다.

클립을 마우스 오른쪽 버튼으로 클릭하면 컨텍스트 메뉴가 열리며, `역재생`, `노멀라이즈`, `음높이 참조 클립으로 변환`, `MIDI 내보내기`, `페이드 커브 유형` 등의 기능이 있습니다. 동일 트랙의 여러 클립을 선택하면 컨텍스트 메뉴에서 `글루`를 선택하여 단일 오디오 클립으로 병합할 수 있습니다.

여러 오디오 클립을 선택한 뒤 컨텍스트 메뉴에서 `그룹`(또는 `G`)을 선택하면 그룹화됩니다. Reaper 또는 VEGAS Pro와 유사하게 같은 그룹의 클립은 편집 시 연동됩니다. 클립 왼쪽 상단의 체인 버튼을 클릭하면 그룹 연동 편집을 일시적으로 비활성화/활성화할 수 있습니다. 그룹화된 클립을 선택하고 `그룹 해제`(또는 `U`)를 선택하면 그룹에서 제거됩니다.

트랙 인터페이스 왼쪽은 트랙 헤더 영역으로, 트랙을 추가/삭제하고 트랙 파라미터를 조정할 수 있습니다. 트랙을 마우스 오른쪽 버튼으로 클릭하여 복제할 수 있습니다.

Reaper와 유사하게 HiFiShifter의 트랙에도 트랙 그룹 개념이 있습니다. 트랙 헤더 영역에서 한 트랙 헤더를 다른 트랙 헤더로 드래그하면 트랙 그룹을 생성할 수 있습니다. 트랙 그룹은 하나의 파라미터 패널을 공유합니다. 실제 사용 시 "한 파트당 하나의 트랙 그룹"으로 정리하는 것이 좋습니다.

트랙 인터페이스의 기능 버튼:

- `BPM`: 프로젝트 전체의 템포 BPM을 조정합니다. HiFiShifter는 현재 가변 BPM을 지원하지 않습니다.
- `매 마디 비트 수`: 프로젝트의 마디당 비트 수를 설정합니다.
- `그리드`: 프로젝트의 그리드 간격을 설정합니다.
- `기준 음계`: 프로젝트 전체의 기준 음계 설정을 조정합니다. 사용자 정의 음계를 허용합니다. 음계 기능은 주로 후술할 `피치 스냅` 및 기타 피치 조정 관련 기능과 관련됩니다.
- `정지` 버튼 및 `재생 / 일시 정지` 버튼: 재생을 제어합니다.
- `파일 브라우저`: HiFiShifter의 파일 브라우저 창을 엽니다.
- `메모장`: HiFiShifter 메모장 창을 열어 Markdown 형식의 텍스트를 기록하고 표시합니다.
- `자동 크로스페이드`: Reaper/VEGAS Pro의 유사 기능과 동일하게, 활성화하면 클립 이동 시 겹치는 부분의 크로스페이드 엔벨로프를 자동 조정합니다.
- `그리드 스냅`: 활성화하면 클립의 모든 조정이 그리드 위치에 스냅되려고 합니다. `Shift`를 누르면 임시로 스냅을 전환할 수 있습니다.
- `재생 헤드 기준 확대/축소`: 마우스 휠로 수평 확대/축소할 때의 중심을 설정합니다. 비활성화 시 마우스 커서 위치가 중심이고, 활성화 시 항상 재생 헤드 위치가 중심입니다(Reaper/VEGAS Pro 로직과 유사).
- `파라미터 에디터에서 재생 헤드 이동 허용`: 비활성화하면 파라미터 에디터의 편집 패널 클릭으로는 재생 헤드가 이동하지 않습니다. 트랙 인터페이스나 파라미터 에디터의 시간 코드 영역을 클릭해야만 재생 헤드가 이동합니다.
- `자동 스크롤`: 활성화하면 재생 중 재생 헤드를 따라 인터페이스가 수평으로 자동 스크롤됩니다.
- `그룹 무시`: 활성화하면 그룹화된 모든 오디오 클립 편집 시 그룹 연동 편집을 전역적으로 무시합니다.

## 4. 파일 브라우저

파일 브라우저를 사용하면 특정 폴더를 열고, 그 안의 오디오 파일을 조건 검색 및 정렬하고, HiFiShifter의 트랙 인터페이스로 드래그 앤 드롭할 수 있습니다. 검색은 정규 표현식을 지원합니다. 오디오 파일을 클릭하면 자동으로 미리듣기가 재생됩니다. `Ctrl` 및 `Shift` 키를 누른 상태에서 파일을 다중 선택할 수 있습니다. 왼쪽 드래그로 하나 이상의 오디오 파일을 시간축에 일괄적으로 추가할 수 있습니다. 오른쪽 드래그로 `시간을 가로질러 추가` / `트랙 간 추가` 메뉴가 나타납니다. `트랙 간 추가`는 여러 오디오 클립을 세로 방향으로 여러 트랙에 걸쳐 시간축에 추가할 수 있습니다.

트랙 영역에 포커스가 있을 때 단축키 `Ctrl + F`를 누르면 빠른 검색 창이 열립니다. 이것은 파일 브라우저 기능의 간소화된 버전으로, 폴더 내 오디오 파일을 빠르게 조건 검색 및 미리듣기하고 시간축에 추가할 수 있습니다.

## 5. 파라미터 에디터

파라미터 에디터는 HiFiShifter의 핵심 기능 중 하나로, 현재 선택된 트랙의 다양한 파라미터를 편집할 수 있습니다.

트랙의 파라미터 편집 기능을 활성화하려면 먼저 해당 트랙의 합성 버튼 `C`를 누르고 오디오 분석이 완료될 때까지 기다려야 합니다. HiFiShifter는 오프라인 렌더링을 사용하며, 파라미터를 편집할 때마다 다시 렌더링이 완료될 때까지 기다린 후에 청취할 수 있습니다.

### 1. 알고리즘 및 파라미터

현재 버전의 HiFiShifter는 다음 3가지 음성 튜닝 알고리즘과 파라미터를 지원합니다.

- **PC-NSF-HiFiGAN**: OpenVPI의 오픈소스 노래 특화 hifigan 보코더로, HiFiShifter의 기본 알고리즘입니다.
    - `피치`: 음성의 피치를 조정합니다.
    - `브레스 게인`: 브레스를 활성화하면 브레스 볼륨을 조정할 수 있습니다(VR-hnsep 모델 기반).
    - `텐션`: 음성의 텐션을 조정합니다.
    - `포먼트 시프트`: 포먼트 시프트를 조정합니다.
    - `볼륨`: 음량을 조정합니다.
- **World**: 오픈소스 고품질 음성 분석 및 합성 알고리즘.
    - `피치`: 음성의 피치를 조정합니다.
- **VsLib**: VocalShifter 공식 음성 분석 및 합성 라이브러리. VsLib 알고리즘은 Windows x86_64 버전에서만 사용 가능합니다.
    - `피치`: 피치를 조정합니다.
    - `볼륨`: 음량을 조정합니다.
    - `팬`: 팬을 조정합니다.
    - `포먼트 시프트`: 포먼트 시프트를 조정합니다.
    - `브레스`: 브레스를 조정합니다.
    - `합성 모드`: 합성 모드를 조정합니다. 특정 모드에서는 일부 파라미터가 무효화될 수 있습니다.
        - `모노`: VocalShifter의 M 알고리즘, 단음 악기용.
        - `모노 (포먼트)`: VocalShifter의 V 알고리즘, 단음 보컬용.
        - `코러스`: VocalShifter의 P 알고리즘, 하모니용.

하나의 트랙은 하나의 알고리즘만 사용할 수 있습니다. 여러 알고리즘을 동시에 사용하려면 별도의 트랙으로 분리하세요.

트랙 그룹은 파라미터를 공유하며, 하위 트랙은 루트 트랙으로부터 파라미터를 상속받습니다. 특히 하위 트랙에는 `센트 차`와 `도수 차`라는 두 가지 추가 파라미터가 있어 루트 트랙에 대한 상대적인 피치 변화를 편리하게 조정할 수 있습니다. `도수 차`는 프로젝트의 음계 설정을 기준으로 합니다.

`선택` 도구로 `피치` 구간을 복사한 후, `센트 차`나 `도수 차`에 붙여넣으면 적절한 오프셋 값이 자동 계산되어 적용됩니다.

### 2. 선택 도구

선택 도구를 사용하면 파라미터 라인 구간을 선택하고, 드래그하거나 마우스 오른쪽 버튼 컨텍스트 메뉴를 통해 파라미터를 조정할 수 있습니다.

자주 사용하는 단축키:

- `Ctrl + C`: 복사
- `Ctrl + V`: 붙여넣기
- `Ctrl + Z`: 실행 취소
- `Ctrl + Y`: 다시 실행
- `Ctrl + A`: 모두 선택
- `Ctrl + R`: 선택 해제
- `BackSpace`: 초기화
- `[` / `]`: 선택 범위 내 파라미터 라인을 아래로/위로 이동

선택한 파라미터 라인을 왼쪽 드래그하면 `드래그 방향` 설정에 따라 수직, 수평 또는 자유롭게 이동할 수 있습니다. 왼쪽 드래그 중에 오른쪽 버튼을 누르면 드래그 방향을 빠르게 전환할 수 있습니다.

선택한 파라미터 라인을 오른쪽 드래그하면 진폭을 조정할 수 있습니다. 위로 드래그할수록 진폭이 커지고 아래로 드래그할수록 작아집니다.

파라미터 에디터 내에서 마우스 오른쪽 버튼을 클릭하면 컨텍스트 메뉴가 열리며, 선택한 파라미터 라인에 대해 `초기화`, `센트 단위로 조바꿈`, `도 단위로 조바꿈`, `값 설정`, `평균화`, `스무딩`, `비브라토 추가`, `퀀타이즈`, `평균 퀀타이즈` 등의 작업을 수행할 수 있습니다.

`Alt` 키를 누르면 선택 범위에 대해 4점 편집 모드가 됩니다. VocalShifter의 동일 유형 기능과 유사하게 4개 점을 드래그하여 현재 파라미터 라인을 휘게 할 수 있습니다.

`Alt` 키를 누른 상태에서 선택 영역 가장자리를 드래그하면 선택 영역 내의 파라미터 라인을 스트레치할 수 있습니다.

### 3. 그리기 도구

그리기 도구를 사용하면 파라미터 라인을 그릴 수 있습니다.

왼쪽 드래그로 `드래그 방향` 설정에 따라 자유롭게 또는 수평으로 파라미터 라인을 그릴 수 있습니다. 왼쪽 드래그 중에 오른쪽 버튼을 누르면 드래그 방향을 빠르게 전환할 수 있습니다.

오른쪽 드래그로 현재 라인을 재설정할 수 있습니다.

### 4. 직선/비브라토 도구

그리기 도구 버튼을 마우스 오른쪽 버튼으로 클릭하면 직선/비브라토 도구로 전환됩니다. 이 도구를 사용하면 직선 또는 비브라토를 그릴 수 있습니다.

왼쪽 드래그로 `드래그 방향` 설정에 따라 자유롭게 또는 수평으로 직선을 그릴 수 있습니다. 왼쪽 드래그 중에 오른쪽 버튼을 누르면 드래그 방향을 빠르게 전환할 수 있습니다.

왼쪽 드래그 중에 마우스 휠을 굴리면 직선에 수평 사인파가 중첩되며, 휠로 진폭을 조정할 수 있습니다. `Alt` 키를 누른 상태에서 휠을 굴리면 주파수를 조정할 수 있습니다. `파라미터 미세 조정` 수정 키(기본값 `Ctrl`)를 누른 상태에서 휠을 굴리면 미세 조정할 수 있습니다.

오른쪽 드래그로 현재 라인을 재설정할 수 있습니다.

`Tab` 키를 눌러 파라미터 에디터의 편집 도구(선택 도구/그리기 계열 도구)를 전환할 수 있습니다.

### 5. 피치 스냅

다양한 편집 도구로 피치 파라미터를 편집할 때 피치 스냅을 사용하면 반음 또는 음계에 스냅할 수 있습니다. `Shift` 키를 누르면 임시로 스냅을 전환할 수 있습니다.

피치 스냅 버튼을 마우스 오른쪽 버튼으로 클릭하면 피치 스냅 설정 메뉴가 열리며, 양자화 단위와 허용 오차를 조정할 수 있습니다.

- `양자화 단위`: `반음`과 `음계` 두 가지 유형이 있습니다. 음계를 선택하면 기준 음계는 프로젝트의 음계 설정입니다.
- `허용 오차`: 스냅의 허용 범위를 조정합니다. 허용 오차 내의 피치 편집은 스냅되지 않고, 범위를 벗어난 편집은 가장 가까운 허용 오차 경계에 스냅됩니다.

예를 들어, 보컬 하모니 효과를 만들려면:

1. 프로젝트 음계를 확인하고 설정합니다.
2. 피치 스냅을 활성화하고 양자화 단위를 `음계`로 설정합니다.
3. 음계 하이라이트 기능을 활성화하여 이동 도수를 쉽게 관찰할 수 있도록 합니다.
4. 선택 도구로 수직 드래그합니다.

또는 하위 트랙의 `센트 차` 및 `도수 차` 파라미터를 사용할 수도 있습니다:

1. 프로젝트 음계를 확인하고 설정합니다.
2. 하모니 트랙의 헤더를 메인 보컬 트랙으로 드래그하여 트랙 그룹을 생성합니다(메인 보컬 = 루트, 하모니 = 하위).
3. 파라미터 에디터를 하모니 트랙의 `도수 차` 파라미터로 전환하고 필요한 도수선을 그립니다. `센트 차` 및 `도수 차` 편집도 피치 스냅을 지원하여 각각 정수 반음 및 정수 도수에 스냅할 수 있습니다.

이렇게 하면 도수 이동에 의한 하모니 효과를 빠르게 만들 수 있습니다.

### 6. 음높이 참조 클립

트랙의 음높이 참조 클립은 타임라인에 피치 곡선을 저장할 수 있는 특수한 클립입니다.

음높이 참조 클립은 다음 경로를 통해 생성할 수 있습니다:

- `파일` 메뉴 또는 파일 드래그를 통해 MIDI를 가져옵니다. MIDI 가져오기 대화상자가 나타납니다.
    - MIDI 파일: 가져올 MIDI 파일을 선택할 수 있습니다. 다른 DAW에서 시스템 클립보드로 출력한 MIDI 데이터 파싱도 지원합니다. 현재 시스템 클립보드를 통한 MIDI 데이터 전송이 확인된 DAW는 Reaper와 FL Studio입니다.
        - Reaper: Reaper의 MIDI 에디터에서 노트를 선택하고 복사하면 선택한 노트 데이터가 시스템 클립보드로 출력되어 HiFiShifter에서 읽을 수 있습니다. Reaper의 클립보드 노트 데이터에는 BPM 정보가 포함되어 있지 않으므로, 가져올 때 현재 프로젝트 BPM을 사용하거나 수동으로 지정할 수 있습니다.
        - FL Studio: FL Studio의 피아노 롤 화면에서 왼쪽 상단의 작은 삼각형을 클릭하여 메뉴를 펼치고 `파일` -> `MIDI 클립보드로 복사`를 선택하면 현재 채널의 모든 노트가 시스템 클립보드로 출력되어 HiFiShifter에서 읽을 수 있습니다.
    - 트랙 선택: 가져올 MIDI 트랙을 선택할 수 있습니다.
    - MIDI BPM을 프로젝트 BPM으로 가져오기: 활성화하면 MIDI의 초기 BPM을 프로젝트 BPM으로 가져옵니다. HiFiShifter는 현재까지도 가변 BPM을 지원하지 않습니다.
    - 노트 BPM: 가져올 노트의 BPM 매핑 방식을 설정할 수 있습니다.
        - MIDI 자체 BPM: BPM 매핑을 하지 않고 MIDI 자체 BPM으로 직접 가져옵니다.
        - 현재 프로젝트 BPM: 노트 BPM을 현재 프로젝트 BPM으로 매핑한 후 가져옵니다.
        - 지정 BPM: 노트 BPM을 수동으로 지정한 BPM으로 매핑한 후 가져옵니다.
    - 멀티트랙 병합: 활성화하면 선택한 모든 트랙과 모든 노트를 자동으로 병합하고, 가장 높은 피치의 노트를 피치 곡선 파라미터로 사용하여 최종적으로 1개의 음높이 참조 클립만 가져옵니다. 비활성화하면 모든 노트가 피치 곡선 파라미터로 가져와질 수 있도록 트랙과 노트를 분할하려고 시도하며, 최종적으로 여러 개의 음높이 참조 클립이 세로로 배치될 수 있습니다.
    - 노트 사이 간격 채우기: 활성화하면 인접한 두 노트 사이의 간격을 자동으로 채웁니다.
- Reaper 프로젝트 또는 Reaper 클립보드를 통해 트랙의 MIDI 아이템을 가져옵니다.
- VocalShifter 프로젝트 또는 VocalShifter 클립보드를 통해 트랙의 MIDI 클립을 가져옵니다.
- 일반 클립을 마우스 오른쪽 버튼으로 클릭하고 컨텍스트 메뉴에서 `음높이 참조 클립으로 변환`을 선택합니다. 이렇게 하면 해당 일반 클립의 원본 피치 곡선이 새로운 음높이 참조 클립의 피치 곡선으로 변환됩니다.
- 파라미터 에디터에서 피치 파라미터를 편집할 때, 선택 도구로 영역을 선택하고 마우스 오른쪽 버튼을 클릭한 다음 컨텍스트 메뉴에서 `음높이 참조 클립으로 저장`을 선택하면 선택 범위 내의 피치 곡선을 새로운 음높이 참조 클립으로 저장할 수 있습니다.

음높이 참조 클립의 일반적인 용도는 다음과 같습니다:

- 트랙에 배치하여 일반 클립이 있는 다른 트랙이 피치를 참조할 수 있도록 합니다. 다른 트랙에서는 파라미터 에디터의 `참조 트랙 그룹` 기능을 사용하여 이 트랙을 보고 피치 곡선을 표시할 수 있습니다.
- 음높이 참조 클립이 트랙 그룹의 루트 트랙에 배치되면, 현재 트랙 그룹의 피치 처리 로직을 변경하여 해당 구간의 원본 피치 곡선을 음높이 참조 클립의 피치 곡선으로 덮어씁니다. 이는 다음 시나리오에 영향을 줍니다:
    - 트랙 그룹 내에서 편집된 적이 없는 피치 곡선 구간이 있으면, 해당 피치 파라미터가 음높이 참조 클립의 피치 곡선으로 직접 덮어쓰기되고 오디오 피치의 재렌더링이 트리거됩니다.
    - `초기화` 관련 기능을 사용할 때——예를 들어 파라미터 에디터에서 그리기 도구로 오른쪽 드래그 작업을 할 때——초기화된 피치 곡선은 트랙 그룹 하위 트랙 내 클립의 원본 피치가 아닌 음높이 참조 클립의 피치 곡선 데이터를 사용합니다.
    - 음높이 참조 클립이 음소거된 경우, 트랙 그룹은 피치를 처리할 때 해당 음높이 참조 클립을 참조하지 않습니다.

음높이 참조 클립을 선택한 뒤 컨텍스트 메뉴에서 `피치 업데이트`를 실행하면 해당 범위의 기존 피치 파라미터가 음높이 참조 클립에 업데이트됩니다.

### 7. 기타 기능

파라미터 에디터의 기타 편리한 기능:

- `클립보드 미리보기`: 선택 도구로 파라미터 라인 데이터를 복사한 후, 선택 영역 내에 클립보드 파라미터 라인이 실시간으로 표시되어 붙여넣기 위치를 파악하는 데 도움이 됩니다.
- `파라미터 값 팝업`: 마우스가 파라미터 라인에 가까워지거나 그리기 편집 작업을 수행할 때 파라미터 값을 표시합니다.
- `파라미터 라인 잠금`: 트랙에서 오디오 클립을 드래그할 때 해당 클립에 대응하는 모든 파라미터 라인도 함께 이동할지 여부를 결정합니다. HiFiShifter의 모든 파라미터 편집 기능은 트랙 기반입니다. 잠기지 않으면 편집된 파라미터 라인이 클립을 따라 이동하지 않습니다.
- `스무딩`: 파라미터 편집 시 자동으로 스무딩을 적용할지 여부와 그 강도.
- `참조 트랙 그룹`: 파라미터가 `피치`일 때 다른 트랙을 선택하여, 다른 트랙 그룹의 피치 곡선을 참조로 피치 편집 창에 표시할 수 있습니다.
- `MIDI 가져오기`: MIDI 파일을 선택하고 단일 또는 다중 트랙의 노트를 피치 라인으로 가져올 수 있습니다.

## 6. 오디오 내보내기

모든 편집을 완료한 후 `파일` 메뉴의 `오디오 내보내기` 기능을 통해 HiFiShifter 프로젝트를 wav 오디오로 내보낼 수 있습니다.

파라미터:

- `내보내기 유형`: `프로젝트` / `분리 트랙`.
- `내보내기 범위`: `전체` / `사용자 지정`. 사용자 지정에서는 시작 및 종료 시간(초)을 설정할 수 있습니다.
- `샘플레이트`: 출력 WAV 오디오의 샘플레이트를 설정합니다.
- `비트 심도`: 출력 WAV 오디오의 비트 심도를 설정합니다.
- `출력 폴더`: 파일 출력 폴더를 설정합니다. 사용 가능한 자리 표시자:
    - `<ProjectFolder>`: 현재 프로젝트가 저장된 폴더. 저장되지 않은 경우 `내 문서` 폴더를 가리킵니다.
    - `<ProjectName>`: 현재 프로젝트의 파일 이름(확장자 제외).
    - 프로젝트 내보내기의 경우 출력 폴더 기본값은 `<ProjectFolder>`입니다. 분리 트랙 내보내기의 경우 기본값은 `<ProjectFolder>/<ProjectName>`입니다.
- `출력 파일 이름`: 파일 출력 이름을 설정합니다. 사용 가능한 자리 표시자:
    - `<ProjectName>`: 현재 프로젝트의 파일 이름(확장자 제외).
    - 기본값은 `<ProjectName>.wav`입니다.
- `분리 트랙 이름 패턴`: 분리 트랙 출력 시 이름 형식을 설정합니다. 사용 가능한 자리 표시자:
    - `<ProjectName>`: 현재 프로젝트의 파일 이름(확장자 제외).
    - `<ExportIndex>`: 내보내기 시 트랙의 순서 번호(0부터 시작).
    - `<TrackIndex>`: 프로젝트 내 트랙 번호(0부터 시작).
    - `<TrackName>`: 프로젝트 내 트랙 이름.
    - `<TrackType>`: 트랙 유형(`Root` / `Sub`).
    - `<TrackId>`: 트랙의 내부 ID(일반 사용자는 사용하지 않는 것이 좋음).
    - 기본 패턴은 `<ExportIndex>_<TrackName>.wav`입니다.
- `분리 트랙 출력 대상` 패널: 내보낼 트랙을 선택할 수 있습니다. 기본적으로 음소거되지 않은 일반 트랙과 루트 트랙만 선택됩니다.
    - 원래 음소거된 트랙을 체크하면 음소거 상태에 관계없이 정상적으로 내보내집니다.
    - 트랙 그룹의 루트 트랙을 체크하면 해당 루트 트랙이 포함된 트랙 그룹이 단일 오디오 파일로 내보내지며, 음소거된 하위 트랙의 오디오 데이터는 제외됩니다.
    - 하위 트랙을 체크하면 해당 하위 트랙이나 해당 루트 트랙의 음소거 상태에 관계없이 정상적으로 내보내집니다.

파일 경로를 입력하는 동안 `자리 표시자` 버튼을 클릭하면 해당 텍스트를 빠르게 입력할 수 있습니다.

위의 모든 파일 경로 관련 문자열은 `%Y-%m-%d-%H-%M-%S`와 같은 시간 형식 문자열을 지원합니다. 출력 경로에 리터럴 `%` 문자를 포함하려면 `%%`를 사용하여 이스케이프 처리하세요.
