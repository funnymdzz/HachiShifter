# HiFiShifter 使用手冊

[简体中文](USERMANUAL.md) | [繁體中文](USERMANUAL_zh-TW.md)| [English](USERMANUAL_en.md) | [日本語](USERMANUAL_ja.md) | [한국어](USERMANUAL_ko.md)

HiFiShifter 是一款圖形化人聲編輯與合成工具。它支援多軌道音訊片段處理，並以軌道組為單位，使用多種聲碼器完成人聲修音、人力調參功能，實現人力 VOCALOID 製作的拼調一體化。

## 一、安裝

下載你的作業系統和架構所對應的 HiFiShifter 安裝包。依作業系統分類，分為 `Windows`、`macOS` 和 `Linux` 這三類。依架構分類，分為 `x86_64` 架構和 `arm64` 架構這兩類。

- Windows 版本提供的是 NSIS 安裝包版（`installer`）和可攜版壓縮包（`portable`）這兩版。對於一般使用者，直接選擇安裝包版並安裝即可。  
  如果你是 Windows 使用者，且不知道 `x86_64` 與 `arm64` 有什麼區別，請選擇 `x86_64` 版本。只有你清楚地了解 `arm64` 是什麼，並且你確實擁有一台 Windows ARM 裝置，則可以下載使用 `arm64` 版。

- macOS 版本提供的是未經過簽名的 dmg 安裝包。因為暫未簽名，安裝時需要經過一個比較複雜的步驟來允許安裝。  
  M 晶片的 macOS 使用者請安裝 `arm64` 版本。只有較老的 Intel 用户需要安裝 `x86_64` 版本。

- Linux 版本提供的是 AppImage 程式包。AppImage 程式包需要在檔案的 `屬性 -> 權限` 中，勾選 `允許將檔案作為程式執行`，之後便可以直接執行。

## 二、功能介紹

軟體的大致操作邏輯和快捷鍵可以參考 Reaper、VocalShifter、VEGAS Pro 等 DAW 軟體。可以透過 `檢視 -> 快捷鍵設定` 來調整自己的快捷鍵偏好。以下介紹均以預設快捷鍵為基礎。

### 1. 選單

`檔案` 選單允許你開啟和儲存 HiFiShifter 專案檔，也允許你進行匯入音訊檔案、匯入 Reaper 專案（`*.rpp`）、匯入 VocalShifter 專案（`*.vshp` 或 `*.vsp`）、匯出音訊等操作。

HiFiShifter 的專案檔副檔名為 `*.hshp` 或 `*.hsp`。除此以外，`另存新檔` 功能還支援將專案檔儲存為明文的 `json` 檔案，或者將當前專案連同使用到的所有媒體檔案，一起打包為歸檔壓縮包 `*.zip`。目前，HiFiShifter 只支援匯入常規的音訊檔案，不支援視訊檔案。

`編輯` 選單允許你進行各種編輯操作。除了比較常規的軌道編輯和參數編輯操作以外，還有 `貼上 Reaper 剪貼簿資料` 和 `貼上 VocalShifter 剪貼簿資料` 這兩項。

- **貼上 Reaper 剪貼簿資料**：當你在 Reaper 中進行物件（Item）複製、軌道複製、MIDI 音符複製操作後，透過該功能，可以將 Reaper 剪貼簿資料快速匯入進 HiFiShifter 中。
    - 物件資料：匯入為 HiFiShifter 中的音符塊，同時允許保留 Reaper 中的調音資料（整體調音和音高包絡均可）。
    - 軌道資料：連同軌道中的物件，匯入為 HiFiShifter 中的軌道和音訊塊。可保留軌道組。
    - MIDI 音符資料：在 HiFiShifter 中使用參數編輯器的 `選擇` 工具選擇一段音高線後，可將 Reaper 剪貼簿 MIDI 音符資料匯入至該段音高線資料。注意，由於 Reaper 剪貼簿 MIDI 音符資料並不包含曲速資料，此時 HiFiShifter 會按照專案 BPM 將其匯入。因此，請在匯入前，確保 HiFiShifter 專案的 BPM 與預期 BPM 相符。

- **貼上 VocalShifter 剪貼簿資料**：當你在 VocalShifter 或者 VocalShifter LE 中進行參數線複製、音訊塊複製、軌道複製操作後，透過該功能，可以將 VocalShifter 剪貼簿資料快速匯入進 HiFiShifter 中。
    - 參數線資料：在 HiFiShifter 中使用參數編輯器的 `選擇` 工具選擇一段參數線後，可將 VocalShifter 剪貼簿參數線資料匯入至該段參數線資料。
    - 音訊塊資料：匯入為 HiFiShifter 中的音符塊，同時允許保留各種參數線資料。
    - 軌道資料：連同軌道中的音訊塊，匯入為 HiFiShifter 中的軌道。注意，HiFiShifter 目前無法區分你上一次複製的內容是音訊塊還是軌道。如果你打算匯入軌道，在 VocalShifter 執行複製軌道操作前，請先保證 VocalShifter 專案中，沒有音訊塊處於選中狀態，這樣才能正常匯入軌道；否則，只有被選中的音訊塊會被匯入。

### 2. 軌道介面

軌道介面是 HiFiShifter 最核心的功能之一，允許你對音訊塊進行裁切、拼接等操作。其操作邏輯，基本可以參考 Reaper。

對於介面操作，拖動滑鼠中鍵（即按住滑鼠滾輪）可以進行介面的平移拖動。水平/垂直的縮放或捲動，可以按住 `Ctrl`、`Alt`、`Shift` 等修飾鍵，並捲動滑鼠滾輪來調整。這些修飾鍵均可在快捷鍵設定中調整。

常用快捷鍵一覽：

- `空格鍵`：播放/暫停（不返回播放起始點）
- `Enter`：播放/停止（返回播放起始點）
- `S`：切割
- `Ctrl + C`：複製
- `Ctrl + V`：貼上
- `Ctrl + Z`：復原
- `Ctrl + Y`：重做
- `Ctrl + A`：全選
- `Ctrl + R`：取消選擇
- `Delete`：刪除音訊塊
- `-` / `=`：對選中的音訊塊的參數線進行下移/上移
- 修飾鍵 `Alt`：按住後，拖動音訊塊開頭/結尾可以拉伸音訊塊，拖動音訊塊中間可以調整音訊塊的內部偏移量
- 修飾鍵 `Shift`：按住後，可以切換是否吸附到格線
- 修飾鍵 `Ctrl`：按住後，拖動音訊塊可以將其複製

音訊塊左上角的 `M` 按鈕可以單獨靜音該音訊塊，小圓圈是音量調整旋鈕。音訊塊左右端分別能夠調整淡入/淡出包絡長度。

對著音訊塊右鍵可以開啟上下文選單，裡邊有 `倒放`、`正規化`、`淡入/淡出包絡類型` 等功能。如果選擇同一軌道的多個音訊塊，在右鍵的上下文選單中，可以對它們進行 `膠合`，合成一個單一的音訊塊。

軌道介面的左側是軌道頭區域，允許你新增或刪除軌道、調整軌道參數等。右鍵軌道可以對軌道進行複製。

類似於 Reaper，HiFiShifter 的軌道也是有軌道組的概念的。在軌道頭區域中，將一個軌道頭拖曳至另一個軌道頭，即可建立軌道組。軌道組共享一套參數面板。實際使用中，建議按「一個聲部一個軌道組」去整理。

軌道介面的一些功能按鈕：

- `BPM`：用於調整專案整體的曲速 BPM。HiFiShifter 目前不支援變 BPM。
- `每小節拍數`：用於設定專案的每小節拍數值。
- `格線`：用於設定專案的格線間距。
- `基準音階`：用於調整專案整體的基準音階設定。允許自訂音階。音階功能主要和後邊會介紹到的 `音高吸附` 以及其他音高調整相關功能有關。
- `停止` 按鈕和 `播放 / 暫停`按鈕：用來控制是否播放。
- `檔案瀏覽器`：用於開啟 HiFiShifter 的檔案瀏覽器視窗。
- `自動交叉淡化`：同 Reaper / VEGAS Pro 的類似功能，啟用後，當移動音訊塊時，如果有音訊塊重疊，自動調整重疊位置的淡入淡出包絡。
- `格線吸附`：啟用後，對音訊塊的所有調整都會嘗試吸附在格線位置。按住 `Shift` 可以進行臨時的吸附切換。
- `基於播放游標的縮放`：捲動滑鼠滾輪可以對介面進行水平縮放。未啟用該設定時，縮放中心為當前滑鼠游標位置；啟用後，水平縮放的邏輯類似於 Reaper / VEGAS Pro 的邏輯，總是基於播放頭位置進行縮放。
- `允許參數編輯器調整播放頭`：停用後，參數編輯器的編輯面板的一些點擊操作不會進行播放頭的調整，只有點擊軌道介面，或者點擊參數編輯器的時間碼區域，才能調整播放頭位置。
- `自動捲動`：啟用後，在播放時，自動對介面進行水平捲動操作，以跟隨播放頭位置。

### 3. 檔案瀏覽器

檔案瀏覽器允許你開啟特定的資料夾，按條件搜尋和排序其中的音訊檔案，並拖曳匯入到 HiFiShifter 的軌道介面。搜尋支援正規表示式。當點擊音訊檔案時，會自動播放試聽。可以按住 `Ctrl` 和 `Shift` 鍵進行檔案的多選。左鍵拖曳檔案可以將一個或多個音訊檔案批次地跨時間加入到時間軸中。右鍵拖曳檔案可以彈出 `跨時間加入` / `跨軌道加入` 的選單。`跨軌道加入` 允許你將多個音訊塊垂直地跨多個軌道加入到時間軸中。

當焦點為軌道區域時，按下快捷鍵 `Ctrl + F` 可以開啟快速搜尋視窗。這個是檔案瀏覽器功能的簡易版，允許你快速按條件搜尋並試聽資料夾內的音訊檔案，最終加入到時間軸上。

### 4. 參數編輯器

參數編輯器是 HiFiShifter 最核心的功能之一，允許你編輯當前選中軌道的各種參數。

要啟用對軌道的參數編輯功能，必須先按下對應軌道的合成按鈕 `C`，並等待音訊分析工作完成後，即可開始進行參數編輯。HiFiShifter 採用離線渲染，每次編輯完參數後，需等待參數重新渲染完畢，才能夠進行試聽。

#### 演算法和參數介紹

目前版本的 HiFiShifter，相容以下 3 種音聲修音演算法和參數：

- **PC-NSF-HiFiGAN**：OpenVPI 開源的歌聲特化的 hifigan 聲碼器，也是 HiFiShifter 預設的演算法。
    - `音高`：允許調整音聲的音高。
    - `氣聲音量`：開啟氣聲後，允許調整音聲的氣聲音量，該參數基於 VR-hnsep 模型。
    - `張力`：允許調整音聲的張力。
    - `共振峰偏移`：允許調整音聲的共振峰偏移。
    - `音量`：允許調整音聲的音量。
- **World**：開放原始碼的高品質語音分析與合成演算法。
    - `音高`：允許調整音聲的音高。
- **VsLib**：VocalShifter 官方發佈的音聲解析與合成庫。VsLib 演算法僅在 Windows x86_64 版本中可用。
    - `音高`：允許調整音聲的音高。
    - `音量`：允許調整音聲的音量。
    - `聲像`：允許調整音聲的聲像。
    - `共振峰偏移`：允許調整音聲的共振峰偏移。
    - `氣聲`：允許調整音聲的氣聲。
    - `合成模式`：允許調整音聲的合成模式演算法，上述某些參數可能會在特定演算法無效。
        - `單音`：VocalShifter 的 M 演算法，單音樂器演算法。
        - `單音+共振峰補正`：VocalShifter 的 V 演算法，單音人聲演算法。
        - `和音`：VocalShifter 的 P 演算法，和聲演算法。

一條軌道只能使用一種演算法，如果想同時使用多種演算法，請分軌。

軌道組共用一套參數，子軌道的各種參數繼承於根軌道。特別地，對於軌道組中的子軌道來說，額外有 `音分差` 與 `度數差` 這兩個參數，可以便捷地調整當前子軌道相對於根軌道的音高變化。其中，`度數差` 參數參考的音階標準是當前專案的音階設定。

使用選擇工具選擇一段 `音高` 參數並複製後，也可以貼到 `音分差` 或者 `度數差` 參數，這樣會自動計算合適的 `音分差` 或者 `度數差` 參數並應用。

#### 編輯工具介紹

##### 選擇工具

選擇工具允許你選擇一段參數線，進行拖曳操作，或者右鍵開啟上下文選單，以進行參數的調整。

常用快捷鍵一覽：

- `Ctrl + C`：複製
- `Ctrl + V`：貼上
- `Ctrl + Z`：復原
- `Ctrl + Y`：重做
- `Ctrl + A`：全選
- `Ctrl + R`：取消選擇
- `BackSpace`：初始化
- `[` / `]`：對選中範圍內的參數線進行下移/上移

左鍵拖曳選中的參數線，可以對其進行垂直、水平或自由的拖曳移動操作。拖曳方向取決於當前的 `拖曳方向` 設定。左鍵拖曳時，按下右鍵，可以快速切換拖曳方向。

右鍵拖曳選中的參數線，可以對其進行幅度的調整，越向上拖曳，幅度越大；越向下拖曳，幅度越小。

右鍵參數編輯器介面內，可以開啟上下文選單，允許對選中的參數線執行各種處理操作，包括 `初始化`、`按指定音分移調`、`按指定度數移調`、`設定到`、`平均化`、`平滑化`、`加入顫音`、`量化`、`均值量化` 等。

按住 `Alt`，可以為選定的參數線進入四點編輯模式。類似於 VocalShifter 中的同類型功能，拖曳 4 個點可以對當前參數線進行彎曲。

按住 `Alt`，拖曳選擇區域邊緣，可以對選擇區域內的參數線進行拉伸。

##### 繪製工具

繪製工具允許你繪製參數線。

左鍵拖曳可以進行自由或者水平的參數線繪製操作。拖曳方向取決於當前的 `拖曳方向` 設定。左鍵拖曳時，按下右鍵，可以快速切換拖曳方向。

右鍵拖曳可以重置當前參數線。

##### 直線/顫音工具

在繪製工具按鈕處右鍵，可以切換至直線/顫音工具。直線/顫音工具允許你繪製直線或者顫音。

左鍵拖曳可以進行自由或者水平的直線繪製操作。拖曳方向取決於當前的 `拖曳方向` 設定。左鍵拖曳時，按下右鍵，可以快速切換拖曳方向。

左鍵拖曳時，捲動滑鼠滾輪，相當於為直線疊加了水平的正弦波，捲動滑鼠滾輪可以調整正弦波的振幅。按住修飾鍵 `Alt` 並捲動滑鼠滾輪，可以調整正弦波的頻率。按住修飾鍵 `參數微調`（預設為`Ctrl`）可以允許你在捲動滑鼠滾輪時進行微調。

右鍵拖曳可以重置當前參數線。

按 `Tab` 鍵，可以切換參數編輯器的編輯工具（選擇工具/繪製類型的工具）。

##### 音高吸附

當使用各種編輯工具編輯音高參數時，音高吸附允許你將音高編輯吸附至半音或者音階。按住 `Shift` 鍵可以臨時切換是否吸附。

對音高吸附按鈕按下右鍵可以進入音高吸附設定選單，允許你調整量化單位和容差。

- `量化單位`：分為半音和音階這兩種類型。其中，當量化單位為音階時，參考的吸附音階為當前專案的音階設定。
- `容差`：允許你調整吸附的容差範圍，容差範圍內的音高編輯，不進行吸附處理；容差範圍外的音高編輯，會被吸附到離它最近的容差邊緣。

比如，如果你想要製作人聲和聲效果，你可以按照以下步驟進行操作：

1. 確認並設定好專案音階；
2. 啟用音高吸附，並將量化單位設定成音階；
3. 啟用音階高亮功能，以便於觀察移調度數；
4. 用選擇工具進行垂直拖曳。

或者，也可以使用軌道組的子軌道功能的 `音分差` 和 `度數差` 參數，按照以下步驟進行操作：

1. 確認並設定好專案音階；
2. 將和聲軌道的軌道頭拖曳到主唱人聲軌道上，組成軌道組，主唱人聲軌道為根軌道，和聲軌道為子軌道；
3. 將參數編輯器切換到和聲軌道的 `度數差` 功能，並按需求繪制度數線。`音分差` 與 `度數差` 參數的編輯也支援音高吸附功能，能分別吸附到整數半音以及整數度數，可以加以利用。

即可快速製作出按度數移調的和聲效果。

##### 其他功能

參數編輯器的其他一些便捷功能：

- `剪貼簿預覽`：使用選擇工具複製參數線資料後，可以在選擇區域內即時展示剪貼簿參數線資料，便於貼上定位。
- `彈出展示參數`：當滑鼠靠近參數線，或者正在執行繪製編輯操作時，展示參數值。
- `鎖定參數線`：在拖動軌道上的音訊塊時，是否同步挪動其音訊塊在軌道上對應的所有參數線。HiFiShifter 的所有參數線編輯功能基於軌道的，如果不鎖定參數線，則已編輯的參數線不會跟隨音訊塊進行移動。
- `平滑度`：編輯參數時，是否自動進行平滑處理，以及自動平滑處理的強度。
- `匯入 MIDI`：允許你選擇 MIDI 檔案，並將其中單軌或多軌的音符匯入為音高線。

### 5. 匯出音訊

做完所有編輯後，透過 `檔案` 選單的 `匯出音訊` 功能，可以將 HiFiShifter 專案匯出為 wav 音訊。

有以下參數：

- `匯出類型`：`專案` / `分軌`。
- `匯出範圍`：`全部` / `自訂`。`自訂` 允許你設定匯出的起始和結束的秒數。
- `取樣率`：允許你設定輸出的 WAV 音訊的取樣率。
- `位元深度`：允許你設定輸出的 WAV 音訊的位元深度。
- `輸出資料夾`：允許你設定檔案輸出的資料夾。允許使用的佔位符如下：
    - `<ProjectFolder>`：表示當前專案所在資料夾。如果當前專案未被儲存過，則指向 `我的文件` 資料夾。
    - `<ProjectName>`：表示當前專案的檔案名稱，不含副檔名。
    - 對於匯出專案來說，`輸出資料夾` 的預設值為 `<ProjectFolder>`；對於匯出分軌來說，`輸出資料夾` 的預設值為 `<ProjectFolder>/<ProjectName>`。
- `輸出檔案名稱`：允許你設定檔案的輸出名稱。允許使用的佔位符如下：
    - `<ProjectName>`：表示當前專案的檔案名稱，不含副檔名。
    - `輸出檔案名稱` 的預設值為 `<ProjectName>.wav`。
- `分軌命名格式`：允許你設定分軌輸出時的命名格式。允許使用的佔位符如下：
    - `<ProjectName>`：表示當前專案的檔案名稱，不含副檔名。
    - `<ExportIndex>`：表示當前分軌在匯出時的順序編號。編號從 `0` 開始。
    - `<TrackIndex>`：表示當前分軌在專案內的編號。編號從 `0` 開始。
    - `<TrackName>`：表示當前分軌在專案內的軌道名稱。
    - `<TrackType>`：表示當前分軌在專案內的軌道類型，根軌道 `Root` / 子軌道 `Sub`。
    - `<TrackId>`：表示當前分軌在專案內的內部 ID。一般使用者不建議使用這個。
    - `分軌命名格式` 的預設值為 `<ExportIndex>_<TrackName>.wav`。
- `分軌選擇面板`：可以選擇你打算匯出的分軌。預設情況下，只會選擇專案內不靜音的普通軌道和根軌道。
    - 如果你勾選了原本處於靜音的軌道，則可以正常匯出該軌道，而忽略該軌道是否處於靜音狀態。
    - 如果你勾選了軌道組的根軌道，則表示將該根軌道所包含的軌道組匯出為單個音訊檔案分軌，且匯出的根軌道的音訊資料中，不包含目前被靜音的子軌道的音訊資料。
    - 如果你勾選了軌道組的子軌道，則可以正常匯出該軌道，而忽略該子軌道或者其根軌道是否處於靜音狀態。

當你正在輸入檔案路徑時，點擊 `佔位符` 那一排按鈕，可以快速將對應文字填入文字框中。

上述所有和檔案路徑有關的字串，均支援時間格式化字串，比如 `%Y-%m-%d-%H-%M-%S`。如果你打算在輸出路徑中包含實際的 `%` 字元，請使用 `%%` 進行跳脫。
